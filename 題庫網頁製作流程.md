## 流程總覽

```
PDF 檔
  ↓ PDF 文字萃取（pdfjs-dist）
  ↓ 文字解析演算法（正則處理）
  ↓ JSON 題庫檔
  ↓ Node.js API（Express + SQLite）
  ↓ 前端網頁（HTML + Vanilla JS）
```

## Step 1｜PDF 文字萃取

**工具：pdfjs-dist**（Node.js，原生可用，無需編譯）

```bash
npm install pdfjs-dist
```

**萃取腳本（explore_pdf.mjs）：**
```javascript
import { getDocument } from 'pdfjs-dist/legacy/build/pdf.mjs';
import { readFileSync } from 'fs';

const data = new Uint8Array(readFileSync(pdfPath));
const pdf = await getDocument({ data, useWorkerFetch: false, isEvalSupported: false, useSystemFonts: true }).promise;

let fullText = '';
for (let i = 1; i <= pdf.numPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    fullText += '\n' + content.items.map(item => item.str).join('');
}
console.log(fullText);
```

> 先跑一次看 PDF 的文字格式，再決定解析策略。

## Step 2｜分析題目格式

不同題庫有不同的格式，需先觀察：

```
格式 A（羅杰晟題庫）：。001題目？①選項1②選項2③選項3④選項4。答案
格式 B：\n001 題目 ... ① A ② B ③ C ④ D
```

**觀察重點：**
- 題號樣式（3位數、4位數、001~999）
- 選項分隔符號（①②③④ vs A. B. C. D.）
- 答案位置（句號後、選項後）
- 哪些字是中文字干擾（目錄、頁碼）

## Step 3｜寫解析腳本

**parse_questions.mjs 核心邏輯：**

```javascript
// 1. 收集所有頁面文字
let fullText = '';
for (let i = 1; i <= pdf.numPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    fullText += '\n' + content.items.map(item => item.str).join('');
}

// 2. 依序拆題（找題號分隔點）
const parts = fullText.split(/(?=\d{3}(?!\d))/);

// 3. 解析單一題目
function parseQuestion(block) {
    const qNum = block.substring(0, 3);
    const optMatches = [...block.matchAll(/([①②③④])([^①②③④]+)/g)].slice(0, 4);
    const options = optMatches.map(m => m[2].trim().replace(/。+$/, ''));
    const firstOptIdx = block.indexOf('①');
    const question = block.substring(3, firstOptIdx).trim();

    // 答案：「④」之後第一個「。」往後
    const after4Idx = block.indexOf('④');
    const after4 = block.substring(after4Idx + 2);
    const dotIdx = after4.indexOf('。');
    const answerRaw = after4.substring(dotIdx + 1).trim().charAt(0);
    const answerMap = { '1': 'A', '2': 'B', '3': 'C', '4': 'D' };
    const answer = answerMap[answerRaw] || answerRaw;

    return { id: qNum, question, options, answer };
}

// 4. 輸出 JSON
const questions = [];
for (const part of parts) {
    const q = parseQuestion(part.trim());
    if (q) questions.push(q);
}
writeFileSync('questions.json', JSON.stringify({ total: questions.length, questions }, null, 2));
```

**輸出格式（建議）：**
```json
{
  "id": "001",
  "question": "題目文字",
  "options": { "A": "選項A", "B": "選項B", "C": "選項C", "D": "選項D" },
  "answer": "D"
}
```

## Step 4｜建立 API Server

**工具：Express + better-sqlite3**

```bash
npm install express better-sqlite3
```

**server.mjs 基本結構：**
```javascript
import express from 'express';
import Database from 'better-sqlite3';
import { readFileSync } from 'fs';

const app = express();
app.use(express.json());

// CORS（開發用）
app.use((req, res, next) => {
    res.header('Access-Control-Allow-Origin', '*');
    res.header('Access-Control-Allow-Headers', 'Content-Type');
    next();
});

// Static files（Serving HTML）
app.use(express.static(__dirname));

// 讀取題庫
const questionsData = JSON.parse(readFileSync('questions.json', 'utf-8'));
const questionMap = {};
for (const q of questionsData.questions) questionMap[q.id] = q;

// ===== API 路由 =====
// GET  /questions       全部題目
// GET  /quiz?count=N    隨機 N 題（不含答案）
// POST /submit          批改答案 { answers: { id: "A" } }
// GET  /wrong-questions 錯題列表
// POST /favorites       新增收藏 { question_id }
// GET  /favorites       收藏列表
// GET  /history         測驗紀錄

app.listen(3000, '0.0.0.0');  // 重要：綁定 0.0.0.0 才可外部連線
```

## Step 5｜SQLite 資料庫設計

```sql
-- 錯題記錄
CREATE TABLE wrong_questions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id TEXT UNIQUE NOT NULL,
    session_id  TEXT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 測驗歷史
CREATE TABLE history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  TEXT,
    total       INTEGER NOT NULL,
    correct     INTEGER NOT NULL,
    score       INTEGER NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 收藏
CREATE TABLE favorites (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id TEXT UNIQUE NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## Step 7｜WSL2 對外暴露 API（Windows 連線）

WSL2 的網路卡是 NAT，對外暴露需要轉發：

### 方法 A：portproxy（推薦，簡單）
在 Windows CMD（系統管理員）執行：
```bash
netsh interface portproxy add v4tov4 listenport=3000 listenaddress=127.0.0.1 connectport=3000 connectaddress=<WSL IPv4>
netsh advfirewall firewall add rule name="WSL API" dir=in action=allow protocol=tcp localport=3000
```

### 方法 B：找 WSL IP 並綁定 0.0.0.0
```bash
ip addr show eth4  # 找 192.168.x.x
# server.js 改為 app.listen(3000, '0.0.0.0')
# Windows 開 Chrome 訪問 http://<WSL_IP>:3000
```

### 方法 C：直接Serving Static（最簡）
把 server.mjs 加上 `app.use(express.static(__dirname))`，
Windows Chrome 直接開 `http://localhost:3000` 即可。

## 常見問題

| 問題 | 解法 |
|------|------|
| `better-sqlite3` 安裝失敗 | 需 Python + make + g++ 編譯器，WSL/Linux 可用 `apt install build-essential python3`，Windows 建議用 [sqlite3 npm](https://www.npmjs.com/package/sqlite3) 或直接用 JSON 檔 |
| `pdfjs-dist` 字體讀不到 | 加 `useSystemFonts: true` 參數 |
| 題目跨頁拼接失敗 | 用 `buffer` 變數保留跨頁殘餘文字，拼完後再處理 |
| 選項結尾殘留答案數字 | 正則替換 `.replace(/。+\d+$/, '')` |
| Express CORS 擋住請求 | 加 `Access-Control-Allow-Origin: *` middleware |
| WSL2 外部連不到 | `app.listen(3000, '0.0.0.0')` + Windows `netsh portproxy` |
| 題庫來自同一本書但分章節編號重複 | JSON 用 `index`（流水號）區分，用 `id` 當外觀題號 |
